# 2026-03-25 Familiar Logo Wiring

## Goal

Use the new familiar logo mark in the main hosted surfaces instead of leaving the product mostly text-branded.

## What Changed

- added a new public SVG logo asset at `public/familiar-mark.svg`
- updated the main app document and static document to use that mark as the favicon
- added the mark to:
  - the homepage hero
  - the docs sidebar brand area

## Why

The product now has a real logo.

The main public surfaces should use that mark consistently while keeping the interface restrained and product-focused.

## Result

The browser tab, homepage, and docs chrome now use the familiar mark.
